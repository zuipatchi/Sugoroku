コードの重複・冗長な実装を検出し、ユーザーに報告してから修正する。

手順:
1. Explore エージェントを spawn して `Assets/Scripts/` 配下の全 `.cs` ファイルを横断的に調査させる
   調査対象：
   - **定数・文字列の重複**: セッション名 `"QuickMatch"` などの文字列リテラルや数値（タイムアウト秒数など）が複数箇所に直書きされていないか
   - **エラーハンドリングの重複**: `catch (Exception e)` + 同じ後処理のパターンが複数メソッドに存在しないか
   - **状態判定の重複**: 同じ `MatchingState` の組み合わせ判定が複数箇所にないか
   - **ロジックの重複**: ほぼ同一の処理フローを持つメソッド（引数が少し違うだけなど）
   各重複について「ファイルパスと行番号（両方）」「重複の種類」「統合の難易度（簡単／要検討）」を報告させる

2. 見つかった重複をカテゴリ・難易度つきで一覧表示し、どれを修正するかユーザーに確認する
   - 難易度「要検討」のものは副作用について議論してから判断する
   - ユーザーが「やめる」と言ったらコミットせず終了する

3. 承認された箇所を修正する
   修正の方針：
   - 定数の重複 → `private const string` または `private static readonly string` に抽出
   - 状態判定の重複 → `private bool IsXxx(MatchingState state)` ヘルパーに抽出
   - ロジックの重複 → `private async UniTask XxxAsync(...)` に抽出して各呼び出し元から呼ぶ
   コーディング規約（CLAUDE.md）を守ること：`_camelCase`、`var` 不使用、アクセス修飾子必須
   動作を変えない。ロジックを変える場合は必ずユーザーに確認する
   過度な抽象化はしない。重複度が低い（パターンが似ているだけ）場合は統合しない

4. 修正内容をコミットする
   コミットメッセージ形式: `重複コードを整理（<変更の要点>）`

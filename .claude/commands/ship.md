変更をレビュー・改善してからコミットし、CLAUDE.mdとdocs/にあるドキュメントとREADME.mdを現状に合わせて更新する。

手順:
1. `git status` と `git diff` で変更ファイルと内容を確認する
2. ドキュメント以外の変更があれば、コードをレビューする
   - 品質・可読性・規約違反・バグの可能性を確認する
   - 問題があれば修正してから次のステップへ進む
   - ドキュメントとコードを見比べて整合性が取れているか確認する
3. 適切なコミットメッセージで日本語でコミットする
4. CLAUDE.md と docs/にあるドキュメント と README.mdを読み、現在のコードベースの状態に合わせて内容を更新する（主要ファイルの場所・アーキテクチャ・実装ロードマップ・ゲームのルールなど）。docs/patterns.md については、今回の変更で追加・変更されたパターン（カード種別・フェーズ・Presenter・EffectType など）がある場合は該当セクションの手順も最新の状態に合わせて更新する
5. 更新したドキュメントをコミットする
